# SITESELECT PRD（MVP-first）

## 1. 背景与目标

在 5 天内完成一个**可交付、可收费**的餐饮选址分析工具最小版本，支持：

1. 本地导入候选点数据
2. 自动评分和排序
3. 输出可直接发给客户的 HTML 报告

### 成功标准（5天）
- 工具可在本地笔记本运行
- 单次分析全流程 < 10 分钟
- 可稳定输出可读报告
- 能支持“99元/次人工咨询交付”

---

## 2. 用户与场景

### 目标用户
- 小餐饮创业者（牛肉饼/卷饼/甜点/烘焙等）
- 小团队选址顾问

### 核心场景
- 用户给出若干候选地址数据
- 工具给出排序与推荐解释
- 顾问人工补充后交付客户

---

## 3. 产品范围

### 3.1 MVP（本次必须）
- CLI 命令
- CSV 输入
- 权重配置（JSON）
- 多维度基础评分
- HTML 报告生成

### 3.2 非MVP（后续）
- Web UI
- 用户账号系统
- 多租户
- 自动化支付
- 高级预测模型

---

## 4. 功能需求

### 4.1 数据输入
- 文件格式：CSV
- 字段：
  - name
  - rent_monthly
  - foot_traffic_index
  - competition_count
  - distance_to_target_m

### 4.2 评分引擎
- 支持权重配置（默认权重 + 用户自定义）
- 指标归一化后加权
- 输出总分（0-100）

### 4.3 报告输出
- 生成单个 HTML 文件
- 包含：
  - TOP 候选点
  - 完整排名表
  - 各指标表现
  - 简要推荐结论

---

## 5. 技术方案（MVP）

- 语言：Python 3.10+
- 依赖：标准库优先（可选轻量模板）
- 存储：本地文件（CSV/JSON/HTML）
- 运行方式：命令行

> 说明：本阶段不引入 PostgreSQL / Redis / Serverless，避免过度工程。

---

## 6. 数据策略

- 主策略：本地数据输入 + 本地缓存
- 增强策略：后续接入高德 API（可选）
- 风险控制：即使外部 API 不可用，MVP 仍可独立运行

---

## 7. 商业化路径

### 7.1 MVP阶段
- 工具开源/免费
- 服务收费：99元/次（人工咨询交付）

### 7.2 后续升级
- 高阶版报告
- API/平台化能力
- 企业订阅

---

## 8. 时间计划（5天）

- Day 1：评分引擎 + 数据格式定稿
- Day 2：CLI + 报告模板
- Day 3：测试 + 示例数据 + 文档
- Day 4：试运行（真实案例）
- Day 5：对外接单与交付优化

---

## 9. 风险与应对

1. **数据质量不齐**：提供字段校验和缺失提示
2. **权重争议**：允许自定义权重并保留默认模板
3. **交付质量波动**：报告自动化 + 人工复核双保险

---

## 10. 验收标准

- `python3 src/siteselect/cli.py analyze ...` 可成功执行
- 输出 `output/report.html`
- 结果可读、排序合理、无崩溃
