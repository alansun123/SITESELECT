# SITESELECT PRD（GUI-first MVP）

## 1. 产品目标

在 5 天内交付一个 **GUI 优先** 的餐饮选址应用，CLI 仅作为辅助入口。

核心要求：
1. 使用者无需命令行也能完整操作
2. 分析过程可解释、可调整
3. 结果可快速导出并用于客户交付
4. 工作页面必须覆盖真实业务录入场景（不是仅展示页）

---

## 2. 用户与场景

### 目标用户
- 餐饮创业者
- 小型选址顾问团队
- 品牌拓店/市场拓展人员

### 典型场景
- 顾问在 GUI 中先定义市场范围与业务上下文
- 再导入候选点并进行竞品/商圈分析
- 调整权重并获得排序
- 预览报告并导出 HTML 给客户

---

## 3. MVP 体验流（Best Practice）

### Step 1 - 项目与市场范围（工作页）
必填业务字段：
- 我方品牌/项目名称
- 友商品牌（可多选）
- 国家
- 城市
- 行政区
- 行业分类

推荐字段：
- 子分类（如“现制茶饮-外带”）
- 门店模型（面积/模型）

系统行为：
- 字段完整性校验
- 支持保存草稿
- 完成后进入下一步

### Step 2 - 候选点导入/提交
- 上传 CSV（后续可扩展 API 抓取）
- 字段校验与错误提示
- 数据预览表格
- 显示数据质量提示（行级）

### Step 3 - 策略配置
- 权重滑块（租金、客流、竞争、距离）
- 一键恢复默认
- 一键归一化（总和 1.00）
- 权重异常提示

### Step 4 - 结果交付
- Top N 推荐
- 完整排名表
- 推荐理由卡片
- 一键导出 HTML 报告

---

## 4. 功能需求

### 4.1 GUI
- 向导式导航
- 关键动作反馈（成功/失败）
- 参数调整即时生效
- 业务字段输入页必须可用（友商品牌/国家/城市/行政区/行业分类）

### 4.2 分析引擎
- 指标归一化
- 加权总分
- 结果排序（稳定可复现）
- 脏数据容错与校验提示

### 4.3 报告
- 页面内预览
- 输出 `output/report.html`
- 包含推荐结论与数据校验说明

### 4.4 CLI（保留）
- `analyze` 命令支持批处理

---

## 5. 技术实现（MVP）

- GUI：Streamlit
- 核心逻辑：Python（复用 `src/siteselect/cli.py`）
- 输入：CSV/JSON
- 输出：HTML

> 说明：优先交互体验和交付效率，不做重后端。

---

## 6. 验收标准

1. `streamlit run app/gui_app.py` 可启动
2. 在 GUI 完成“业务字段录入 → 导入候选点 → 调权重 → 查看结果 → 导出报告”全流程
3. Step 1 可提交友商品牌、国家、城市、行政区、行业分类等字段
4. 报告可在浏览器打开且信息完整
5. CLI 仍可运行

---

## 7. 里程碑（5天）

- Day 1：GUI 框架 + 业务字段工作页（Step1）
- Day 2：候选点导入与校验（Step2）+ 评分配置（Step3）
- Day 3：结果页与报告导出（Step4）
- Day 4：真实样本试跑 + 交互文案修订
- Day 5：演示与客户交付模板
