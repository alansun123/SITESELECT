# SITESELECT GUI-first MVP QA Checklist

## 1) 发布前基础检查（P0）
- [ ] 依赖可安装：`streamlit` 可用，CLI 可执行。
- [ ] GUI 可启动：`streamlit run app/gui_app.py` 无启动错误。
- [ ] CLI 可运行 happy path 并生成 HTML 报告。
- [ ] `output/` 可写，报告文件可落盘。

## 2) GUI 主流程检查（P0）
- [ ] Step1 项目信息可填写并保存在 session。
- [ ] Step2 可上传 CSV，导入成功时展示表格和条数。
- [ ] Step3 权重可调，显示总和，恢复默认可用。
- [ ] Step4 能看到 Top1 推荐、排名明细、可导出报告。

## 3) 数据校验与容错（P0/P1）
- [ ] 空 CSV 给出明确错误提示。
- [ ] 缺失必填字段给出明确错误提示（GUI/CLI一致）。
- [ ] 非法数值（字符串、空值）有处理策略（报错或告警，不可静默污染结果）。
- [ ] 极端权重（总和>>1、负数）有约束或清晰提示。

## 4) 输出质量（P1）
- [ ] 报告包含生成时间、Top N 列表、完整排名表。
- [ ] 排名与分数可复现（同输入同结果）。
- [ ] 中文显示正常，无乱码。

## 5) 回归与可发布判断（P0）
- [ ] Happy path 全绿。
- [ ] P0 缺陷（崩溃/阻断）= 0。
- [ ] 已知限制已记录在发布说明中。
